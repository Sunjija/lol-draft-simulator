## 완료한 일

### 사용 흐름

- 샘플 명단과 챔프폭은 유지하면서 신규 사용자가 `팀 설정 -> 챔프폭 -> 밴픽` 순서로 진행하도록 온보딩을 연결함.
- 앞 단계를 확인하기 전 다음 탭으로 건너뛰지 못하게 하고, 기존 밴픽 기록이 있는 저장 데이터는 온보딩에 다시 갇히지 않도록 호환 처리함.
- 이름, 라인, 티어 입력은 별도 적용 버튼 없이 즉시 상태에 반영됨.

### 챔프폭 관리

- 팀과 선수를 먼저 선택하고 한 선수의 챔프폭에 집중하는 master-detail 구조로 정리함.
- `챔프폭 편집`을 주요 CTA로 강조함.
- `전체/탑/정글/미드/원딜/서폿` 필터, 챔피언 이름 검색, 초상화 클릭 추가 기능을 구현함.
- 긴 브라우저 기본 자동완성 목록 대신 높이가 제한된 챔피언 탐색 그리드를 사용함.
- 챔피언을 추가하거나 숙련도를 변경해도 편집창이 유지됨.
- 숙련도 `장인/잘함/보통/가능`을 색상과 점수로 구분함.
- 이름 기반 빠른 추가와 여러 줄 빠른 입력 기능은 유지함.

### 밴픽 화면

- 20턴 순서 표시를 클릭 가능한 것처럼 보이던 버튼에서 비상호작용 타임라인으로 변경함.
- 블루/레드는 색으로, 밴/픽은 `B/R + 밴/픽` 텍스트로 구분하며 현재 턴은 금색 테두리로 표시함.
- 오른쪽 현재 턴 추천 후보를 가로 3칸에서 세로 카드로 변경함.
- 추천 카드에서 챔피언명, 담당 선수와 라인, 추천 점수를 분리해 표시함.
- 추천 계산, 챔프폭 제한, 밴픽 순서, 조합 및 메타 점수 로직은 이번 UI 작업에서 변경하지 않음.

### 타이포와 레이아웃

- 데스크톱 앱에 공통 타이포 스케일을 적용함.
- 데스크톱 작업 화면에서 표시되는 텍스트는 모두 `13px` 이상임.
- 일반 본문과 안내는 주로 `14px`, 버튼과 입력은 `15px`, 주요 패널 제목은 `18px` 기준임.
- 폰트 확대와 함께 슬롯, 추천 카드, 컨트롤, 선수 선택 행의 높이와 여백도 조정함.
- 랜딩 페이지와 모바일 전용 레이아웃은 별도 규칙을 유지함.

## 남은 일

- 현재 알려진 차단 버그는 없음.
- 실제 신규 사용자에게 다시 사용시켜 팀 설정, 챔프폭 편집, 첫 밴 적용까지의 탐색 시간을 확인할 필요가 있음.
- 모바일은 기존 최적화가 유지되지만 이번 데스크톱 타이포 확대와 동일한 기준으로 재평가하지 않았음.
- 랜딩 페이지 타이포는 이번 작업 범위에서 변경하지 않았음.
- 추천 점수의 실전 적합성은 UI와 별개로 실제 내전 또는 격전 결과를 이용해 지속 검증해야 함.

## 주의사항

- API 키, 토큰, 개인 계정 정보는 저장소나 이 문서에 기록하지 말 것.
- 공개 배포에서는 수기 챔프폭과 정적 데이터만으로 핵심 기능이 동작해야 함.
- Riot API 자동 분석이 필요할 때만 로컬 백엔드를 사용하고, 브라우저 코드에 키를 넣지 말 것.
- 사용자 상태는 브라우저 `localStorage`에 저장되므로 다른 컴퓨터나 다른 도메인으로 자동 이동하지 않음.
- `index.html`에 UI, 상태, 추천 로직이 많이 모여 있어 수정 범위가 넓다. 관련 없는 계산식이나 데이터 파일은 함께 정리하지 말 것.
- `data/`의 메타 파일과 자동 갱신 스크립트는 UI 변경과 소유 범위가 다르므로 필요하지 않으면 건드리지 말 것.
- 작업 트리가 dirty일 경우 기존 변경을 되돌리지 말고 먼저 diff를 확인할 것.

## 다른 컴퓨터에서 시작하기

```powershell
git clone https://github.com/Sunjija/lol-draft-simulator.git
cd lol-draft-simulator
git checkout main
git pull --ff-only origin main
python -m http.server 8899 --bind 127.0.0.1
```

브라우저에서 `http://127.0.0.1:8899/index.html?app=1`을 연다.

Riot API를 사용하는 로컬 기능까지 확인할 때만 별도로 `python server.py`를 실행한다. API 키는 로컬 환경에서만 입력하고 커밋하지 않는다.
